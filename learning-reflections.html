Course Learning Reflections

Throughout this course, I have gained deep insights into algorithmic design and problem-solving techniques, which I can apply in real-world scenarios. Here are some of the key reflections:

1. BRTS Ticket Verification

  • Dynamic Data Handling: Learned to use a vector to store and manage passenger ticket details dynamically, ensuring scalability for fluctuating data.
  • Formatted Output: Implemented structured and readable output using formatting tools like setw for clear data presentation.
  • Real-time Input: Gained experience in capturing and validating user input in a loop, simulating real-time ticket scanning and storage.
  • Input Handling: Learned to handle edge cases with cin.ignore to avoid input stream issues during data entry.

2. Ticket Management and Retrieval System

  • Data Organization: Designed the Passenger class to encapsulate ticket details such as name, source, destination, and ticket price, making data structured and manageable.
  • Display Mechanism: Implemented a method to display stored ticket data, focusing on concise and clear representation.
  • Iterative Access: Used loops to iterate over the vector and retrieve all stored tickets, simulating a basic search or retrieval mechanism.
